# 🔔 Alert System — Cosmic Watch

## Overview

The Alert System proactively notifies users about significant asteroid events based on configurable thresholds. Alerts are generated by background scheduled jobs and delivered in real-time via Socket.io or viewed on the alerts dashboard.

---

## Alert Types

| Type                 | Trigger                                          | Default Threshold | Severity         |
| -------------------- | ------------------------------------------------ | ----------------- | ---------------- |
| `close_approach`     | Asteroid approaches Earth within distance        | 0.05 AU           | warning          |
| `hazardous_detected` | New PHA detected in today's feed                 | Always            | danger           |
| `watchlist_update`   | Watched asteroid has new close approach data     | Always            | info             |
| `risk_threshold`     | Any asteroid exceeds user's risk score threshold | Score ≥ 60        | warning-critical |

## Alert Generation Flow

```
┌─────────────────────────────────────────────────────────────┐
│               CRON JOB: alertCheck.job.js                   │
│               Schedule: Every 30 minutes                    │
│                                                             │
│  1. Fetch upcoming asteroid data (next 7 days)              │
│     └── From MongoDB cache (or NASA API if stale)           │
│                                                             │
│  2. Get all users with alert preferences enabled            │
│     └── Query users collection                              │
│                                                             │
│  3. For each user, evaluate thresholds:                     │
│     ├── Distance threshold vs asteroid miss distance        │
│     ├── Risk score threshold vs calculated risk             │
│     ├── Hazardous flag check                                │
│     └── Watchlist match check                               │
│                                                             │
│  4. Generate Alert documents for triggered thresholds       │
│     └── Avoid duplicates (check existing alerts)            │
│                                                             │
│  5. Save alerts to MongoDB                                  │
│                                                             │
│  6. Push to online users via Socket.io                      │
│     └── io.to(`user:${userId}`).emit('notification:new')    │
└─────────────────────────────────────────────────────────────┘
```

## User Alert Preferences

Users configure their alert thresholds in their profile:

```json
{
  "alertPreferences": {
    "distanceThreshold": 0.05, // AU — alert if asteroid < this
    "riskScoreThreshold": 60, // 0-100 — alert if score > this
    "notifyOnHazardous": true, // Alert on any PHA detection
    "notifyOnWatchlist": true // Alert when watched asteroids update
  }
}
```

## Real-Time Delivery

```javascript
// When alert is generated, push to online users
const pushAlertToUser = (io, userId, alert) => {
  io.to(`user:${userId}`).emit("notification:new", {
    _id: alert._id,
    type: alert.type,
    title: alert.title,
    message: alert.message,
    severity: alert.severity,
    asteroidName: alert.asteroidName,
    createdAt: alert.createdAt,
  });
};
```

## Alert Severity Levels

| Severity   | Color            | Icon | When Used                                |
| ---------- | ---------------- | ---- | ---------------------------------------- |
| `info`     | Blue `#3b82f6`   | ℹ️   | Watchlist updates, routine data          |
| `warning`  | Yellow `#feca57` | ⚠️   | Close approaches, moderate risk          |
| `danger`   | Orange `#ff9f43` | 🔶   | Hazardous asteroid detected              |
| `critical` | Red `#ff4757`    | 🔴   | Critical risk score, very close approach |

## Duplicate Prevention

Before creating an alert, the system checks:

```javascript
const existingAlert = await Alert.findOne({
  userId,
  asteroidId,
  type,
  "data.closeApproachDate": approachDate,
  createdAt: { $gte: new Date(Date.now() - 24 * 60 * 60 * 1000) }, // last 24 hrs
});

if (!existingAlert) {
  await Alert.create(alertData);
}
```
